Insights

二、两地学生需求共性深度提炼

通过 81 份日志跟踪研究，识别出 6 条跨地区需求共性，是个人知识库产品设计的核心依据

1

🎓 「学不懂」是首要矛盾 → 知识讲解是最高频刚需

两地学生课堂实际理解率远低于及格线。SSA 学生课堂只理解 20-60%，EE1 学生因纯口述 + 手写模式，课后信息损耗严重。

关键洞察：现有 AI 给的是 "summary"，学生要的是 "tutorial"。这是个人知识库最核心的价值差异点。

统一诉求：逐知识点展开讲解 + 举例 + 出题测试
2

📁 学习资料四散、格式混杂 → 统一归集 + 智能整理是刚需

SSA 资料来源
  • WhatsApp 群 / 自发送
  • 手机下载 + 拍照截图
  • Google Drive
  • 大量手写体 / 模糊拍照
EE1 资料来源
  • 纸质笔记拍照
  • 电子文档 + 截图
  • 教师私域材料

共性问题：无整理习惯 + 手机端为主 + 非标格式多

统一诉求：多渠道资料一键导入 → AI 自动按课程 / 章节归类 → 关键词搜索定位

两地均有此痛点，需统一解决
3

📝 考试高压驱动学习行为 → 备考是峰值场景

考试模式
  • CAT / CA / quizzes
  • Final Exam（占 60-70%）
  • 过往试题是核心资产
考试模式
  • 高频随堂抽查
  • 期末 3 周内考完 5-7 门
  • 口试（EE1特色）

统一诉求：上传学习资料 → AI 圈定考试范围 → 生成重点提纲 + 模拟题 + 解析 → 根据剩余时间动态调整复习策略

峰值场景：考试密集期 = AI 使用高峰期
4

🧠 AI 输出太泛、不贴合 → 缺乏三层 Context

Context 层SSA 表现EE1 表现
课程 Context没拿到学生课程资料，回答无针对性无法对齐本地教材和教师要求
学生个体 Context基础差，需用更简单语言 + 分步讲解需匹配学生能力水平
学校 / 教师 Context作业格式、考试要求、查重标准教师模板、格式规范极严格、口试要求

关键判断：知识库的价值不在于存储，而在于让 AI 的每次输出都基于这三层上下文。

Context 层是"好用"与"能用"的分水岭
5

✍ 写作是长链路痛点 → 需要工具链而非单点功能

痛点链路：审题（不知道scope）→ 找文献（来源可信度）→ 结构框架（逻辑论证展开）→ 学术表达（语言润色）→ 引用格式（规范对齐）→ 查重+AI检测

审题（不知道 scope） 找文献（来源可信度） 结构框架（逻辑论证） 学术表达（语言润色） 引用格式（规范对齐） 查重 + AI 检测

统一诉求：不是要一个"帮我写"的按钮，而是覆盖写作全链路的辅助工具，且输出必须符合本地学术规范。

6

🌎 本地化缺失 → 通用 AI 不懂"我们这儿的规矩"

本地化需求
  • 肯尼亚 / 尼日利亚本地案例
  • 本地现实场景举例
  • 贴合本地考试局考纲
本地化需求
  • 符合EE1写作规范
  • 教师私域材料对齐
  • 口试逻辑和流程

统一诉求：AI 需要"懂本地"，不仅是语言层面的本地化，更是教育制度、考核标准、学术规范层面的本地化。

Decisions

六、关键产品决策建议

基于用研洞察提炼出的战略性产品决策，直接影响资源投入和方向选择

1
决策1：先做深度，后做广度

通用 AI 已经覆盖了广度。个人知识库的差异化价值在于基于用户资料的深度理解。先把「基于课程资料的深度讲解」做到极致，再拓展其他场景。

2
决策2：移动端优先，资料导入体验是生死线

SSA学生几乎全在手机端操作。如果导入体验不顺畅，知识库底座就建立不起来，上层AI能力再好也没用。必须支持 WhatsApp/Telegram 一键转发入知识库。

3
决策3：Context层是做好用还是能用的分水岭

学生不满的不是AI不够聪明，而是AI不懂他们的课、不懂他们的老师、不懂他们的考试。Context 配置层的完善程度，直接决定产品的口碑传播能力。

4
决策4：不做两套产品，通过配置适配两地差异

底层能力完全统一，通过 Context 配置层适配不同地区。SSA 和 EE1 的差异是配置差异，不是产品差异。这决定了研发效率和维护成本。

Summary

七、总结

两地 81 名学生的日志跟踪研究揭示了一个清晰的图景：

学生的核心需求不是在AI时代"学得更多"，而是在资源匮乏、教育体系僵化的现实中"学得懂、考得过"。

个人知识库的价值主张

📚
1. 底座层
把散落的学习资料变成结构化的个人知识体系
🧠
2. Context 层
让AI基于你的课程、你的水平、你的考试来输出
🚀
3. 应用层
覆盖讲解、备考、写作三大高频场景
✨ 这不是做一个"更好的 ChatGPT"，而是做一个
懂你的课、懂你的考试、懂你哪里不会的学习伙伴。
